SLIDE 03 · GAP

你想做的 vs 一个人能做的

🎯 Jarvis（标杆）

虚构，钢铁侠世界观

  • 有形机器（钢铁侠装甲、HUD、霍兰桥实验室）
  • 多模态实时感知（视觉/雷达/红外/工厂数据）
  • 能直接控制硬件（武器、飞行、防御网）
  • 多线程并发任务，多年项目记忆
  • 有人设、幽默、对人情境敏感
  • 背后是漫威 + Marvel Studios + 几十个 VFX 团队

🛠️ Nexus（现实可达）

独立开发者 · 2026 LLM 时代

  • 软件桌面陪伴，能听能说能记得你
  • 语音 + 文字双通道，5 语言
  • 本地优先，BYO-LLM
  • 长期记忆 + 关系等级 + 情绪建模
  • 叙事 artifact（信、年度纪念册、多日线）
  • 整个项目目前是你一个人在推
差距不是你能力问题——Jarvis 那种"在场 + 主动性 + 多模态 + 真做事"需要硬件 / OS 层集成，不是更强模型能解决的。物理事实，不是失败。

SLIDE 11 · OPEN QUESTIONS

4 个等你拍板的事（按你方便）

  1. 代码签名预算？
    $220/年 vs 永远 unsigned。"no commercialization" 是不是包括"no recurring infra cost"？
  2. 危机响应触发时伴侣破角色 vs 留在角色里？
    破角色（"我担心你，请联系 988…"）安全；带着 hotline 留在角色里更 artifact。这是个产品调性决定。
  3. 默认 TTS 切换 Voxtral / Kyutai 的时机？
    opt-in 两个 patch 后 flip default？还是永远 opt-in 不动 default？
  4. self-tune 阈值跑偏怎么 reset？
    隐藏的"reset to defaults" 按钮 vs 更显眼的安全阀。但你要的是 silent，所以隐藏更一致。
SLIDE 12 · MY RECOMMENDATION

我会按这个顺序推

为什么这个顺序

  • 1.1 先：8 月 EU 是硬截止；实现成本最小（regex + LLM + 5 语言 hotline list）；做完就解锁可以放心给更多人用。
  • 1.2 紧跟："普通用户能装"是 1.1 的同伴问题；做完一次性投入解决多年。
  • Tier 2 三件事任选：都是机会、不急，按你心情。我会先选 2.1 阈值自调（写好的引擎需要消费数据，闭环优先）。
  • Tier 3 机会主义：触到那块代码再修，不要为了清债而清债。

不推荐做的：12 个月 Jarvis-er 那一摊远景。**等 polish 期跑完再说**。强行追"完整 Jarvis"会让 polish 散掉。
