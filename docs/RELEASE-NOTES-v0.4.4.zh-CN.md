# Nexus v0.4.4 — Beta 反馈与文案调优

状态：草稿。Klein 明确要求最终发布检查、tag 和 GitHub Release 之前，不要发布。

v0.4.4 继续保持 0.4 系列的草稿评审状态：当前公开稳定版是 v0.4.3，
v0.4.4 仍为草稿。这个切片不新增感知来源、不新增通知、
不生成动态文案，也不做桌宠行为；它只让 beta 反馈更安全地比较，并让 check-in
文案更不容易滑向监控感。

## 变化

- Beta 反馈只保留在结构化 issue template 中，不再随应用发布反馈归一化或存储 helper。
- 在 beta validation issue template 里增加结构化 v0.4 字段。
- 为每个桌面陪伴 check-in 原因增加五语言文案护栏。
- 增加固定安全 fallback 文案，避免未来某条文案不安全时让 in-app check-in 静默消失。
- Nexus 不在应用内保留包含原始评论、时间戳、窗口标题、剪贴板正文、截图、文件路径、
  用户 ID 或会话轨迹的反馈载荷；issue 是否提交仍由用户控制。
- 反馈标签不会用于训练、个性化、评分、自适应行为或持久分析。

## 不包含

- 暂不发布正式 v0.4.4。
- 不改 package 版本号。
- 不打 tag，不创建 GitHub Release。
- 不切换 README 稳定版入口。
- 不做反馈分析引擎、自适应文案系统、A/B 测试、外部通知、消息发送、工具执行、效率评分、
  新感知来源、桌宠移动或桌面窗口控制。
