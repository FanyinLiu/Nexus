# Nexus v0.3.5 中文说明

> 这版的主题是：**让 Nexus 更稳、更安全，也让记忆更透明。**
>
> 它不是把 Nexus 做成替你干活的 Codex 式智能体，而是继续把它收敛成一个本地优先、能陪伴、能记住、可控制的桌面伙伴。

## 发布状态

- 代码版本已经是 `0.3.5`。
- 当前分支已经准备好 0.3.5 内容。
- 正式 GitHub Release 还需要在合并后创建 `v0.3.5` tag，并跑发布工作流。

## 你能感受到的变化

### 首次启动更容易跑通

- 配 Ollama、DeepSeek 或 OpenAI-compatible 服务时，Nexus 会先做本地预检。
- 常见问题会给明确修复建议，比如 Ollama 地址、模型名、是否需要 `ollama pull qwen3:8b`。
- 安全的内置修复只会改 Base URL 和模型名，不会碰 API Key。
- 首次启动会显示“5 分钟内完成首次对话”的状态。
- 设置里可以导出本地首次运行 QA 报告，不包含聊天内容、模型输出、API Key 或密钥。

### 发布和更新状态更清楚

- 新增发布信任检查，明确 macOS、Windows、Linux 的签名和更新状态。
- 现在未签名的 macOS 包不会假装能安全自动更新，只会检查新版本并打开下载页。
- SQLite 初始化和打包冒烟已经纳入发布验证。

### IPC、权限、密钥和审计更硬

- 多个高风险 IPC 入口增加了请求校验。
- 文件、桌面上下文、外部动作、插件、MCP、vault 等路径增加了元数据审计。
- 审计不会记录正文、完整路径、命令内容、密钥、token 或敏感参数。
- renderer 仍然拿不到明文 API Key，继续用 vault ref 方式走主进程。

### 聊天记录开始进入 SQLite 迁移预演

- 现有聊天仍然以 localStorage 为权威来源，不会被删除。
- 新增隐藏 dry-run，可以只看统计信息，不读取/发送正文。
- 新增隐藏迁移、回滚、运行时镜像和元数据对账能力。
- 这些入口默认关闭，需要 feature flag 和确认，不会自动改用户数据。

### 记忆变得更透明

- Memory 设置页会显示记忆是否启用、长期记忆数量、每日日志数量和当前存储边界。
- 用户可以暂停全部记忆，也可以暂停单条长期记忆，不需要删除。
- 回复里如果用到了记忆，会出现来源提示。
- 展开来源提示后，可以看到当前对应的记忆预览、缺失状态或暂停状态。
- 从回复进入 Memory 页时，会高亮这次回复用到的记忆和日志片段。
- 新增长期记忆、旧版记忆、每日日志的只读 dry-run，为以后迁移到 SQLite 做准备。

## 没有做什么

- 没有新增 Codex 式自动工作智能体。
- 没有新增自动执行任务。
- 没有把长期记忆真正迁移到 SQLite。
- 没有删除现有 localStorage 数据。
- 没有把 API Key、记忆正文、聊天正文暴露给 renderer 或审计日志。

## 仍然已知的限制

- macOS 正式可信自动更新仍需要 Developer ID 签名和公证。
- Windows 签名也还需要真实签名凭据。
- macOS 本地 smoke 包仍是 ad-hoc/未签名状态，Gatekeeper 可能保留 quarantine 标记；首次打开可能需要右键打开，熟悉命令行的用户也可以用 `xattr` 移除 quarantine。
- Windows 最终可信安装仍依赖真实签名证书；未签名安装包可能出现 SmartScreen 提示。
- 可选语音模型、KWS、SenseVoice、Python 侧能力缺失时会降级，不影响主程序启动。

## 下一步建议

0.3.5 的重点是底层信任和白盒记忆。下一阶段更适合转向 **视觉存在感 / 桌面头像状态机 / Live2D 可选渲染入口**，而不是先扩工具框架或多智能体。
