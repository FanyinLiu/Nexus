# Nexus v0.3.6 — 0.3 的地基收尾

> **基础设施收尾版。** 这版把 Nexus 0.3 线收束到安全、记忆、设置、可见边界和桌面感知入口。它不把 0.4 的真正桌面陪伴感知硬塞进小版本里。

## 用户能看到什么变化

### 设置页更清楚

- 白天和暖白主题下，设置卡片、开关文字、底部按钮和版本入口都更容易读。
- 控件标题和说明文字统一成更紧凑的大小，设置页不再忽大忽小。
- 没有未读通知时，主面板不再显示空的通知摘要卡片。

### 桌面感知入口变得可见

Memory 页的“上下文感知”现在会显示三项状态：

- 当前窗口上下文
- 剪贴板上下文
- 屏幕文字 OCR

每一项都会显示已启用、可开启、未启用或不可用。页面也明确写出隐私边界：Nexus 只把脱敏后的文本摘要交给聊天上下文，不传原始截图。

### 0.3 线正式收尾

0.3 已经把后续成长需要的地基补齐：

- 安全和错误脱敏
- 记忆可见、可暂停、可追溯
- 模型、伙伴、记忆、语音、工具、主动陪伴的设置入口
- 桌面上下文隐私检查
- 发布和打包验证

所以下一个真正的大体验，不应该继续塞进 0.3.x。

## 这版不做什么

- 不做常驻屏幕录制。
- 不做自动控制鼠标键盘。
- 不做桌宠跟随鼠标。
- 不新增 Codex 式工作智能体。
- 不做“时间流逝后的主动陪伴”主循环。

后续方向应该拆开：

- **0.4.0：桌面陪伴感知。** Nexus 安静知道用户离开 Nexus 后在电脑上经历了一段时间，并在合适节点粗粒度关心。
- **0.5.0：桌面宠物行为。** 桌宠根据鼠标、打字、停顿和桌面状态做动作反馈，但不能挡住用户。

## 发布前重点验证

- 白天、暖白、夜晚主题下设置页都可读。
- Memory -> 上下文感知能看到当前窗口、剪贴板、OCR 三项状态。
- 三项状态滚动到对应区域时不会被底部保存栏挡住。
- 桌面上下文隐私检查仍然通过。
- 发布对齐测试已经指向 v0.3.6。

## 已知限制

- macOS 本地 smoke 包仍是未签名构建，Gatekeeper 可能保留 quarantine
  标记。首次启动可能需要右键打开，进阶用户也可以用 `xattr` 移除
  quarantine。
- Windows 安装器信任仍依赖真实签名证书；未签名构建可能出现
  SmartScreen 提示。
- 0.4.0 的桌面陪伴感知主循环没有包含在本次发布里。
