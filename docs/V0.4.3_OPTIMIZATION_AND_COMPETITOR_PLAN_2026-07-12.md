# Nexus v0.4.3 — Companion Surface Cohesion & Transparency

Status: historical implementation and review plan for the now-stable
`v0.4.3` release. The current public stable release is `v0.4.3`; `v0.4.4`
remains a draft until Klein explicitly authorizes a later release workflow.
This plan does not authorize publishing any later draft.

## Release Promise

v0.4.3 makes Nexus feel like one quiet desktop companion instead of several UI
systems placed beside one another.

- Live2D is the Panel's visual subject and remains visually first.
- Voice remains a core capability, but the Panel has no persistent voice button
  or voice action. Settings → Voice manages start, stop, and cancel; Frameless
  and Pet surfaces retain their direct voice entry.
- The current reply stays a temporary caption; full history stays in the
  user-opened conversation sheet.
- Settings Home and every active settings page use one full-height shell, one
  semantic color system, one spacing rhythm, and one scroll owner.
- Memory and desktop awareness say plainly what Nexus can read, what may be sent
  to a configured model, what is kept briefly on this device, and what Clear
  removes.
- Pause retains recent companion-continuity data. Turning awareness off changes
  only the draft; recent data is purged only after the user saves the enabled to
  disabled transition. Discarding the draft never purges it.

Success is not more UI. It is fewer visible concepts, calmer hierarchy, and
more truthful boundaries.

## Must

### M1. One full-height, color-consistent Settings system

- **User value:** Home, Letters, Memory, Voice, and every other active page feel
  like the same product and occupy the same shell instead of ending halfway.
- **Code:** `src/app/styles/settings-visual-system.css`,
  `src/features/uiV2/settings-v2.css`,
  `src/features/settingsV3/settings-v3.css`, and the active V2 shell.
- **Contract:** use existing `--nx-settings-*` semantic roles for ink, muted
  text, surface, soft surface, field, hover, border, accent, success, warning,
  and danger. Do not add another final/override stylesheet and do not remove
  legacy imports in this slice.
- **Exit:** Settings Home and all active V3 pages fill the same shell; Home
  cards use remaining height without becoming a dashboard; only the active
  content region scrolls.

### M2. Honest Memory and desktop-awareness disclosure

- **User value:** users can distinguish quiet companion continuity from richer
  desktop sources before enabling them.
- **Code:** `src/features/settingsV3/MemorySectionV3.tsx`, the five locale files,
  and the existing transparency view model.
- **Contract:** put companion continuity and desktop-awareness sources directly
  after the Memory status block, before long-term-memory editing. Keep advanced
  source configuration folded.
- **Truth boundary:** window information, clipboard text, and OCR results may be
  sent with the current request to the configured chat model and used in its
  reply. When VLM is enabled, the current screenshot is sent to the configured
  vision-model endpoint; the screenshot itself is not added to the chat prompt,
  but its analysis may be used in the current reply. The local boundary describes
  three short-lived categories: a coarse companion summary, the latest local
  check-in decision, and minimal session/lifecycle/expiry metadata.
- **Privacy:** render no raw window title, clipboard body, screenshot, file path,
  exact timer, or raw signal key.

### M3. Correct save, pause, and clear semantics

- **User value:** trying a switch cannot silently delete recent state.
- **Code:** `src/components/SettingsDrawer.tsx`,
  `src/components/settingsSaveEffects.ts`, and
  `src/features/settingsV3/MemorySectionV3.tsx`.
- **Contract:** toggling awareness off changes only the draft. Discard restores
  the committed value and retains recent summary/decision. Saving a committed
  `true` to draft `false` transition clears both. Pause retains both. Explicit
  Clear deletes both.
- **Exit:** pure save-boundary tests and a real renderer lifecycle test prove
  the complete sequence.

### M4. Active-route truth

- **User value:** audits protect the screens people actually open.
- **Contract:** supported Live2D, sprite, and VTube Studio configurations use
  `CompanionPanelV2`; active Memory settings load `MemorySectionV3` through the
  V3 section registry. The legacy routes remain explicit fallback only.
- **Exit:** route/alignment tests fail if either active import is replaced by a
  retired surface.

### M5. Preserve the Live2D-first, voice-capable companion surface

- Do not change the existing conversation prompt copy or interaction in this
  slice.
- Do not add a greeting panel, dashboard, second waveform, permanent status
  card, or automatic conversation-sheet opening.
- Keep Live2D as the Panel's visual subject. Do not add a persistent voice
  action to the Panel. Start, stop, and cancel belong in Settings → Voice;
  Frameless and Pet retain their direct voice entry.

### M6. Keep weather and time truthful but non-persistent

- The active V2 companion path does not poll ambient weather or run legacy
  scene clocks. It adds no persistent clock, weather card, particles, or
  full-window scene.
- Tools Settings is the single weather entry: one enable switch and one manual
  default city. Device time stays local; Nexus does not read GPS; a city is
  sent to OpenStreetMap Nominatim and Open-Meteo for an explicit weather
  request, a user-created weather reminder, or opt-in legacy ambient weather.
- The per-request local-time reminder includes local date, weekday, hour,
  minute, and the device time-zone name so spoken time answers remain grounded
  without adding visible UI.

#### Weather localization contract

- `settings.uiLanguage` now travels as a locale through the weather lookup request,
  IPC validation, tool registry, ambient-weather hook, cache key, reminder
  speech path, and assistant fallback builders.
- Provider results remain locale-neutral where possible: stable WMO condition
  codes, numeric measurements, and dates. Nexus localizes
  conditions, errors, period labels, summaries, cards, and fallback TTS at the
  presentation boundary in all five supported UI languages.
- The result-card period-prefix parser remains compatible with archived Chinese
  `今天` / `明天` summaries while accepting the equivalent five-language
  prefixes, so older persisted tool results still render without duplicated
  day labels.

## Should

- Keep transient captions bounded and readable across warm, day, night, and
  black themes without introducing another glass-card system.
- Keep Live2D loading lazy, teardown idempotent, hidden-window ticker pausable,
  and unsupported gestures gracefully degradable.
- Replace source-only assertions with runtime/DOM evidence whenever a setting
  can be mounted, expanded, and exercised in the renderer.
- Keep Settings controls at the existing accessible hit target and focus-ring
  contract in desktop, narrow, portrait, and short-height layouts.

## Defer

These expand behavior beyond the v0.4.3 cohesion release:

- background marketplace, generated scenes, upload library, and per-background
  blur/opacity editors;
- mouse following, typing following, desktop wandering, autonomous movement,
  window hopping, and multi-character scenes;
- welcome/marketing pages, activity dashboards, relationship scoring, streaks,
  and cross-session proactive check-in history;
- new sensing sources, continuous camera/screen observation, raw activity
  timelines, and richer gesture authoring;
- legacy panel deletion or wholesale legacy Settings CSS removal before active
  coverage proves it safe.

## User-Perceived Change

After this slice, a user should notice four things:

1. Opening Settings always shows one complete, balanced panel; no child page
   appears to occupy only half of it.
2. Home and child pages share the same warm/day/night/black color roles, borders,
   fields, focus treatment, and density.
3. Memory explains “companion continuity” separately from “desktop awareness,”
   including the chat-model and vision-model boundary in plain language.
4. Turning desktop awareness off is safe to try: Cancel preserves recent data;
   Save removes it; Pause preserves it; explicit Clear removes it immediately.

The companion stage itself should not feel newly busier.

## Three Primary Competitor Conclusions

### Kindroid: conversation should feel deliberate

Kindroid is the experience yardstick: obvious voice control where the surface
calls for it, natural interruption, optional history, and inspectable memory
choices. Nexus should borrow that clarity without putting a persistent voice
action back on the Panel, and not its call-page skin, subscription breadth,
selfies, or video-avatar scope.

### AIRI: embodiment should stay modular

AIRI is the architecture yardstick: voice, avatar, memory, provider, and desktop
runtimes remain separable; heavy systems stay lazy; failures are observable.
Nexus should borrow that modularity, not AIRI's game/plugin breadth or developer
console density.

### Desktop Mate: co-presence should remain quiet

Desktop Mate is the desktop-form yardstick: character-first scale, predictable
utilities, and low-interruption presence. Nexus should borrow the restraint, not
mouse chasing, window hopping, alarms, or autonomous movement in v0.4.3.

Together these references support Nexus's position:

> A restrained local-first Live2D desktop companion whose voice state, memory,
> and desktop-awareness boundaries remain visible and understandable.
