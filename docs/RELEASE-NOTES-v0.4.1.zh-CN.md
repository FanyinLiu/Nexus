# Nexus v0.4.1 — 粗略时间语言

> **草稿。** 这个小版本继续沿着 0.4 的桌面陪伴感知路线走：只调整
> Nexus 如何表达“时间过去了”，不新增感知来源、不做主动关心扩张，也不进入
> 0.5 的桌宠跟随鼠标、打字或窗口控制。
>
> 暂不发布。只有在用户明确要求最终发布检查、tag 和 GitHub Release 时，才进入
> 正式发布流程。

## 变化

### 时间表达有了独立安全层

陪伴时间桶文案、五语言映射、精确时间泄漏检测和安全兜底格式化现在集中在
专门的陪伴时间语言模块里。安静观察核心仍然只负责观察规则和时间桶计算。

这让 0.4.1 的边界很清楚：只改变“怎么说时间”，不改变“怎么观察桌面”。

### 多语言精确时间泄漏会被拦住

Nexus 之前已经会避免 `37 minutes`、`90 seconds` 这类英文精确计时。这个草稿
继续拦截中文、日文、韩文里带阿拉伯数字的精确时长，例如：

- `1小时30分钟`
- `1時間30分`
- `2시간 10분`

如果某个异常或精确的 elapsed label 走到显示或 prompt 边界，Nexus 会退回对应
语言的安全粗粒度时间桶文案。

## 仍然不做

这个版本不包含：

- 主动关心调度
- 新桌面感知来源
- 活动历史 UI
- 生产力评分
- 跟随鼠标、跟随打字、桌宠窗口控制
- 跨会话累计时间

这些继续留给后续 0.4.x 或 0.5。

## 验证重点

发布前应覆盖：

- 时间桶边界
- 五语言时间桶文案
- 英文精确时间禁用
- 中文、日文、韩文数字时长禁用
- prompt 对异常精确标签的安全兜底
- recent summary 对精确标签的拒绝
- 透明 UI 对精确标签的安全兜底
