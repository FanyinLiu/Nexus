# Nexus v0.4.3 — 用户可见透明度

状态：草稿。Klein 明确要求最终发布检查、tag 和 GitHub Release 之前，不要发布。

v0.4.3 继续沿着桌面陪伴感知路线往前走：v0.4.0 是稳定的安静观察地基，
v0.4.1 是粗略时间语言草稿，v0.4.2 是 Check-In 策略草稿。这个切片只做
设置里的透明度说明，让用户更清楚 Nexus 能观察什么、什么会进入模型、什么只在
本地短暂保存。

## 变化

- 为设置里的陪伴感知透明度区块增加确定性的视图模型。
- 把透明度说明拆成三行：可观察、会进入模型、本地存储。
- 明确暴露模型不可达原因，以及清理近期摘要按钮不可用的本地原因。
- 增加本地主动陪伴原因模型，让设置可以解释 Nexus 正在保持安静，还是可以轻声出现，
  同时不暴露原始 signal key。
- 增加可见的近期粗略陪伴摘要行和隐私边界行，让用户知道本地存在什么，但不会看到原始
  桌面内容。
- 把聊天桌面上下文链路接到最近一次本地主动陪伴决策记录上，所以正在聊天时可以解释
  为什么主动陪伴保持安静。
- 继续把存储生命周期限制在短生命周期会话状态：近期摘要和安全的近期主动陪伴决策都会在暂停或关闭感知时清理。
- 保留 `rawContentVisible: false` 这条硬边界。
- 增加轻量不变量检查，避免状态文案、行顺序、清理按钮状态、主动陪伴原因、粗略时间语言和原始内容边界在后续重构中静默漂移。

暂停陪伴感知会阻止摘要继续进入模型；但如果本地已经有近期摘要，清理它仍然可用，
因为这是删除本地陪伴数据。

## 隐私边界

透明度视图模型不得携带原始窗口标题、截图、剪贴板正文、消息正文、文件路径、
精确到分钟/秒的时间表达，或桌面活动时间线。

主动陪伴透明度模型只暴露粗粒度原因组，例如正在聊天、安静时段、专注保护、冷却、
重复抑制，或可以轻声出现；不携带原始窗口标题或原始 signal key。

设置页保存的近期主动陪伴决策是临时状态，只包含 `reason`、`surface`、`priority`
和布尔值 `signalKeyPresent`；不会持久化原始 signal key。

近期摘要行只显示粗略时间标签和宽泛活动标签。隐私边界行是固定说明，保证截图、
剪贴板正文、精确计时、文件路径和消息正文都不会进入 UI 透明度契约。

当前运行时写入器是被动的：它只记录供设置透明度使用的决策，不会发出主动消息、
通知或对话。
