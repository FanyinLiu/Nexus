## 变化

- 为设置里的陪伴感知透明度区块增加确定性的视图模型。
- 在真实启用的 Memory Settings V3 路由渲染完整透明度契约，不再依赖已退役的旧设置区块充当审计证据。
- 把透明度说明拆成三行：可观察、会进入模型、本地存储。
- 明确暴露模型不可达原因，以及清理近期摘要按钮不可用的本地原因。
- 增加本地主动陪伴原因模型，让设置可以解释 Nexus 正在保持安静，还是可以轻声出现，
  同时不暴露原始 signal key。
- 增加可见的近期粗略陪伴摘要行和隐私边界行，让用户知道本地存在什么，但不会看到原始
  桌面内容。
- 主 Panel 不再常驻语音按钮；设置 → 语音负责保存、开始、停止和取消，Frameless 与 Pet
  仍保留直接语音入口。
- 三个内置 Live2D 模型是 Mao、Haru 和 Hiyori。本次发行记录了 smoke、透明边缘和首帧证据。
- 把聊天桌面上下文链路接到最近一次本地主动陪伴决策记录上，所以正在聊天时可以解释
  为什么主动陪伴保持安静。
- 继续把存储生命周期限制在短生命周期会话状态：暂停只停止采集和模型可达，粗粒度近期陪伴摘要、最近一次本地主动陪伴决策，以及最少的会话/生命周期/过期标记会保留，供用户查看或明确清理；关闭感知时仍会清理近期陪伴数据。
- 保留 `rawContentVisible: false` 这条硬边界。
- 增加轻量不变量检查，避免状态文案、行顺序、清理按钮状态、主动陪伴原因、粗略时间语言和原始内容边界在后续重构中静默漂移。
- 继续让真实启用的 V2 界面以语音和 Live2D 为主：该路径不再启动旧环境天气轮询或场景时钟。
  天气只在工具设置中保留一个手动城市入口，并明确说明不读取 GPS、何时联网；主动询问天气、执行用户创建的天气提醒，或启用旧版环境天气时会联网，本地时间则始终不联网，时间提醒包含设备时区名称。
- 天气条件、查询错误、当前与逐日摘要、结果卡片时段、助手摘要和兜底语音现已统一支持简体中文、繁体中文、英语、日语和韩语。

## 未签名发行契约

官方 GitHub Releases 是唯一支持的二进制下载来源。v0.4.3 面向 macOS
arm64、Windows x64 与 Linux x64；不宣称提供 macOS x64 或 universal 资产。

### macOS unsigned auto-update limitation

macOS arm64 应用采用 ad-hoc 签名，不是 Apple Developer ID 签名或公证。
ad-hoc 不等于 Apple 信任，Gatekeeper 仍可能要求右键打开或明确移除隔离属性。
应用只检查新版本并打开官方 release 页面；用户需要手动下载并替换应用。

### Windows unsigned installer limitation

Windows x64 NSIS 安装器状态为 `NotSigned`。SmartScreen 可能显示未知发布者，
安装器无法提供发布者身份验证或稳定声誉。用户应先确认资产来自官方 GitHub
Release，再决定是否继续运行。

每个平台构建都在同一个 GitHub Release 中附带独立校验清单：
`SHA256SUMS-windows.txt`、`SHA256SUMS-macos.txt` 和
`SHA256SUMS-linux.txt`。Linux 用户只下载其中一种包格式时，可以运行
`sha256sum --ignore-missing -c SHA256SUMS-linux.txt` 进行校验。

暂停陪伴感知会停止采集，并阻止摘要继续进入模型，但不会静默删除已有的短生命周期
本地状态。如果本地已有近期陪伴数据，用户仍可明确清理；关闭感知时则仍会清理摘要和决策。

## 隐私边界

透明度视图模型不得携带原始窗口标题、截图、剪贴板正文、消息正文、文件路径、
精确到分钟/秒的时间表达，或桌面活动时间线。

主动陪伴透明度模型只暴露粗粒度原因组，例如正在聊天、安静时段、专注保护、冷却、
重复抑制，或可以轻声出现；不携带原始窗口标题或原始 signal key。

设置页保存的近期主动陪伴决策是短生命周期的本地决定，并带有最少的会话、生命周期和
过期标记；信号是否存在只保留为布尔决定细节，不会持久化原始 signal key。

近期摘要行只显示粗略时间标签和宽泛活动标签。隐私边界行是固定说明，保证截图、
剪贴板正文、精确计时、文件路径和消息正文都不会进入 UI 透明度契约。

当前运行时写入器是被动的：它只记录供设置透明度使用的决策，不会发出主动消息、
通知或对话。
