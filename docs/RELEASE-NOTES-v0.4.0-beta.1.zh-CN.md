# Nexus v0.4.0-beta.1 — 桌面陪伴感知开始

> **Beta。** 这是 v0.4 桌面陪伴感知的第一个 beta。它开始建立
> “打开 Nexus 后，用户在别处工作时，Nexus 能安静理解时间流逝”的循环。
> 这不是稳定版发布，主要用于验证时机、语气、隐私边界和系统权限体验。

## 用户会感受到什么

### Nexus 可以安静地保持陪伴连续性

当桌面上下文感知开启，且 Nexus 打开但用户没有直接和 Nexus 交互时，
Nexus 可以形成一段短期、粗粒度的陪伴摘要：

- 用户最近没有和 Nexus 互动
- 大致活动类型，例如 coding、reading、browsing、idle 或 frequent switching
- 粗略时间，例如“一会儿”“半小时左右”“一小时左右”
- 是否应该保持安静，因为用户可能正在专注

这不是监控，也不是效率评分。目标是让之后的对话或低频关心更有连续感。

### 时间说法刻意保持粗略

v0.4 使用陪伴式时间说法：

- 刚开始
- 一会儿
- 半小时左右
- 一小时左右
- 两小时以上

Nexus 不应该像秒表一样说话。面向用户的陪伴文案不精确到分钟或秒。

### 关心策略会保守触发

第一版 check-in 策略只处理强信号：

- 长时间连续活动
- 频繁切换
- 活动后长时间 idle
- 用户离开一段时间后回到 Nexus

它会尊重暂停、冷却时间、专注状态和免打扰时段。Beta 阶段重点验证这些
时机听起来是关心、太早、太晚，还是太频繁。

### 设置页说明隐私边界

Memory 设置页会说明桌面陪伴感知：

- Nexus 可以观察什么
- 它保存什么
- 什么会进入模型上下文
- 如何暂停
- 如何清理近期陪伴摘要

可见摘要只保留粗粒度信息，不展示原始桌面内容。
暂停桌面陪伴感知或关闭桌面上下文感知时，Nexus 会清理近期本地陪伴摘要，
避免设置页继续显示旧的感知状态。

## 隐私边界

允许：

- 脱敏后的陪伴摘要
- 大致活动类别
- 粗略时间桶
- 短期近期摘要元数据

不允许：

- 原始截图
- 完整 OCR 内容
- 完整剪贴板内容
- 私人消息正文
- 私人文件路径
- 精确到秒的计时
- 隐藏活动日志

## 本 beta 不包含什么

- 不包含 v0.5 的桌宠鼠标跟随或打字反应。
- 不自动控制鼠标或键盘。
- 不在没有明确用户动作时读取文件或发送消息。
- 不提供效率仪表盘或原始活动时间线。
- 不新增 Codex 式自主工作 agent 行为。

## Beta 验证重点

请使用 GitHub **Beta Validation Report** 模板，并尽量说明：

- Nexus 是否打开但没有被使用
- 你是否正在主动和 Nexus 聊天
- 当时大致在做什么桌面活动
- 时间说法是否自然
- check-in 是否像关心、冷淡、唠叨或监控
- 暂停和清理近期摘要是否容易找到
- 系统权限流程是否令人困惑

不要附上原始截图、剪贴板内容、私人窗口标题、私人消息正文或私人文件路径，
除非它们是你专门创建的安全测试素材。

## 已知问题

- 仍需要在 macOS、Windows 和 Linux 的权限流程中做真实 beta 验证。
- 多语言文案可能需要根据 beta 反馈继续调整。
- v0.5 桌宠桌面行为会等这条感知循环稳定可信后再推进。
